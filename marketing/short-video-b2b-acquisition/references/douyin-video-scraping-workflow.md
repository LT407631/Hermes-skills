# 抖音竞争对手视频文案自动化提取工作流

> 适用场景：通过 CDP 登录抖音后，批量提取竞争对手视频的完整标题/描述、话题标签、AI 章节要点、评论区引导
> 
> 2026-05-09 实战验证，成功率约 80%（部分视频 URL 会 301 重定向到其他视频）

---

## 整体流程

```
搜索关键词 → 采集搜索结果中的视频 URL → 并行访问各视频页 → 提取文案
       ↑                                            ↓
   按账号/视频维度整理                            产出竞品文案汇总
```

## 步骤 1：搜索定位（浏览器模式）

从搜索关键词开始，通过 `browser_navigate` 进入抖音搜索结果页：

```python
# 搜索视频结果，URL 编码关键词
browser_navigate(url=f"https://www.douyin.com/search/{quote(keyword)}?type=video")

# 需要 scroll 一次才能触发结果加载
browser_scroll(direction="down")
```

**搜索技巧：**
- **视频搜索** (`?type=video`)：看到视频标题、播放量、发布时间、Hashtag
- **用户搜索** (`?type=user`)：看到账号粉丝数、获赞数、简介
- **最佳实践：两个维度各搜一次，交叉比对**

## 步骤 2：从搜索结果页提取视频 URL

这是关键步骤。抖音 SPA 页面不直接暴露 URL，需要用 `browser_console` 提取：

```javascript
// 提取所有视频链接 + 对应的文本描述
let results = [];
document.querySelectorAll('a[href*="/video/"]').forEach(a => {
  let text = a.textContent.trim().substring(0, 80);
  results.push({url: a.href, text: text});
});
// 去重
let seen = new Set();
results.filter(r => { if(seen.has(r.url)) return false; seen.add(r.url); return true; });
```

**输出示例：**
```json
[
  {"text": "01:3326设计师必学！4步方案讲解法...", "url": "https://www.douyin.com/video/7616962053832812746"},
  {"text": "02:05584全屋定制高定门店签单模式...", "url": "https://www.douyin.com/video/7554379712827591971"},
]
```

**注意：** URL 文本中的播放数字（如 `26`、`584`）就是该视频的播放量。

## 步骤 3：并行访问视频页面（核心技巧）

使用 `delegate_task` 将每个视频 URL 分配给独立的子代理并行处理。每个子代理的任务相同：

### 子代理任务模板

```json
{
  "context": "账号：@XXX\n视频描述（从搜索结果获取的标题）：XXXX\nURL: https://www.douyin.com/video/XXXXXX\n\n用浏览器打开这个视频页面，等待加载完成后，提取页面上所有可见的描述文字、话题标签和章节要点内容。返回你找到的所有文本。",
  "goal": "访问抖音视频，提取完整描述文案",
  "toolsets": ["browser"]
}
```

### 子代理提取的内容

每个子代理应提取以下字段：

| 字段 | 提取方法 | 说明 |
|------|---------|------|
| 标题/描述 | `browser_console` 提取页面文本 | 视频发布时写的文案 |
| 话题标签 | 从标题中提取 `#...` 部分 | 5-10个不等 |
| 章节要点 | 页面上的"章节要点"折叠区域 | AI 自动生成的视频内容摘要，分段+时间戳 |
| 评论区引导 | 评论区顶部或作者置顶评论 | 常见格式「留言 XXX 发你资料」 |
| 账号信息 | 页面上的账号名称、粉丝数、获赞数 | 用于综合评估 |
| 发布时间 | 页面显示的发布时间 | 判断时效性 |
| 互动数据 | 点赞、评论、收藏、分享数 | 评估爆款程度 |

### 章节要点提取示例

抖音的"章节要点"功能是 AI 自动生成的视频内容分段，是最接近完整文案结构的数据：

```
【章节要点完整版】
| 时间点 | 章节标题 | 内容摘要 |
|--------|---------|---------|
| 00:00 | 引言 | 开篇引入 |
| 00:17 | 第1步：前期准备 | 形象/工具/店面体验 |
| 00:28 | 第2步：进店拦截 | 一句话一个动作让客户进来 |
| ... | ... | ... |
| 01:40 | 结语 | 签单靠完整流程 |
```

## 步骤 4：结果汇总与呈现

所有子代理返回后，按账号分组整理，格式如下：

```
## ⭐ ① @账号名（关键信息）
### 视频1：标题（发布时间 · 播放量）
```
📝 章节骨架：
① 步骤1 → 要点
② 步骤2 → 要点
...
🏷️ #标签1 #标签2
💬 评论区引导：「留言XXX发资料」
```

### 视频2：标题
...
```

## 已知问题与应对

### 问题 1：视频 URL 跳转/重定向

**现象：** 某些视频 URL 访问后 301 到另一个完全不相关的视频（常见于较老的视频或已删除的视频）。

**应对：**
- 访问后检查 URL 是否变化（`browser_snapshot` 中的 URL）
- 如果跳转了，记录：`{original_url} → {redirected_url} (内容不同)`
- 在汇总中标注该视频无法获取

### 问题 2：章节要点无法通过 DOM 提取

**现象：** 章节要点在无障碍树中可见但标准 DOM 提取不到。

**应对（按优先级）：**
1. 尝试 `document.querySelector('[class*="chapter"]')` 或包含"章节"的关键字查找
2. 从 `browser_snapshot` 返回的文本树中搜索"章节要点"文本
3. 如果都没有，使用 `browser_vision` 截图识别（但该功能当前经常失败）

### 问题 3：delegate_task 超时

**现象：** 子代理运行时间过长（抖音页面加载慢）。

**应对：**
- 每次并行最多 3 个任务（max_concurrent_children=3）
- 分批处理（第一批3个，第二批3个...）
- 如果某个任务长时间无响应，直接跳过

### 问题 4：搜索页面无结果

**现象：** 搜索结果页显示"登录后即可搜索更多精彩视频"

**应对：**
- 用户在 Chrome 中手动登录抖音
- 从用户主页重新搜索（保持登录态）
- 如果仍不行，使用 `browser_navigate` 直接进搜索页（有时 SPA 路由比搜索框更稳定）

---

## 效率优化建议

1. **全量提取 URL 后再分批处理** — 不要逐个搜索，先一次性把所有视频 URL 提取出来
2. **每个 delegate_task 只处理 1 个视频** — 简单任务更快完成
3. **跳过播放量过低的视频**（< 20 播放）— 节省资源
4. **优先处理 1 个月内发布的视频** — 时效性对二次创作更重要
5. **同账号的视频可以集中到一个 delegate_task** — 减少重复登录开销
