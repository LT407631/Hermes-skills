# B端概念解释脚本模板

> 适用于向门店/工作室老板解释行业术语或概念。
> 结构经腾哥多次修正，已验证通过。

## 结构

```
① 不是什么（排比否定）
② 是什么（用销冠/导购的具体场景）
③ 用精准数字的生活类比
④ 一句话总结
```

## 示例（SOP）

```
SOP不是什么高大上的东西。

你店里那个销冠，
客户说"我回去考虑考虑"，
他知道今天下午发一条报价对比，
明天上午十点打个电话聊方案细节，
后天发段安装视频强化信任。

你把他这三天的跟单步骤写下来，
换个人照做，也不会跟丢。

这跟做番茄炒鸡蛋一个道理。
普通人做菜，全凭感觉。
SOP告诉你——
番茄切3厘米的块，
鸡蛋打3个，
油温烧到160℃，
先炒蛋，30秒出锅，
再炒番茄，2分钟出汁，
合在一起翻5下。

每一步的时间、温度、大小，全部写清楚。
换谁来炒，都是那个味道。

SOP就是把最牛那个人怎么干的，
写成谁都能照着干的东西。
```

## 关键规则

- 开头不要「打比方」，先用「不是什么」否定错误认知
- 例子必须精确到具体数字（温度/时间/次数/尺寸）
- 类比用厨房场景（番茄炒鸡蛋），不用行业内部例子
- 主角用「销冠」不用「老师傅」（B端老板关心销售）
- 每条控制在20-25秒口播
- 收尾用一句话总结，不画蛇添足
