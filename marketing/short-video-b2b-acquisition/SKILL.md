# 全屋定制 B端短视频获客运营

## 适用场景

全屋定制工厂/门店/品牌方，通过抖音、视频号等短视频平台，向以下 B 端客户引流获客：

- 全屋定制工作室
- 全屋定制门店
- 装饰公司
- 设计师工作室

**核心转化路径：** 内容引流 → 获取客资（微信/电话）→ 详细沟通 → 促成合作

**排除路径：** 小黄车带货、直播带货（当前验证对 B 端优质客资效果有限）

---

## 交付标准（严格遵守）

1. **绝不废话** — 直接给成品文案、脚本、标题。禁止多余寒暄、铺垫、废话
2. **绝不编造** — 不懂直说「该内容暂无可靠参考」。严禁编数据、编案例、编行业规则
3. **直接交付** — 腾哥下任务，直接出完整成品。不反问、不反复确认、不拆分闲聊
4. **信息不足时** — 只简洁提示「缺少 XX 信息，补充后直接给成品」，不绕弯子
5. **逻辑清晰** — 分点精简，重点突出，方便直接复制使用
6. **客观真实** — 流量逻辑、平台规则只说真实的，不臆测趋势

## 内容类型矩阵

### 1. 工厂实力展示型
- 工厂实拍、生产线、板材仓库、设备演示
- 目的：建立信任，展示交付能力
- 话术重点：规模、设备、产能、品控

### 2. 专业背书型
- 设计师培训、行业干货、工艺解析
- 目的：塑造专业形象，吸引工作室/设计师
- 话术重点：设计理念、工艺细节、行业趋势

### 3. 案例展示型
- 落地案例、实景拍摄、前后对比
- 目的：展示效果，降低客户决策成本
- 话术重点：项目信息、设计亮点、客户反馈

### 4. 合作政策型
- 招商政策、合作模式、扶持体系
- 目的：直接转化，引导私信留资
- 话术重点：政策要点、利润空间、支持力度

### 5. 行业话题/痛点型
- 行业乱象、避坑指南、经营痛点
- 目的：引发共鸣，筛选精准客户
- 话术重点：痛点提炼、解决方案、引导互动

| 规则 | 标准 | 背景 |

### 视觉风格

- **Theme-Next/Gemini 风格** — 浅灰背景 `#f5f5f5`，白色卡片无圆角，干净简约
- **卡片阴影** — 轻（`0 1px 3px rgba(0,0,0,0.04)`），不喧宾夺主
- **边框** — 细（`1px solid #eee`），分隔清晰
- **演示卡片/手机模拟** — 竖屏优先（9:16），用 `phone-frame` 类模拟手机效果
- **流程图** — 用 emoji 图标 + 箭头展示流转，比文本标签好看（如 `📋→💬→💰`）
- **支持暗色模式** — CSS 变量 + 切换按钮（快捷键 T）

### 内容结构

- **标题 → 简介 → 分块卡片**，每块一个 `h2`
- **表格优先于文字段落** — 对比信息、参数列表用表格呈现
- **列表用 ul/ol**，每项一行，不写长段落
- **价格/数据用醒目卡片**（`price-card featured` 高亮方案）
- **手机看板/效果预览**用模拟边框居中展示

### 不用场景

- ❌ 不写大段无结构文字
- ❌ 不用小于 12px 的字体
- ❌ 不用外文衬线字体（如 Lato）
- ❌ 不做宽屏布局（腾哥会认为「看着怪怪的」）

| | Obsidian | QMD (@tobilu/qmd) | Felo LiveDoc |
|---|---|---|---|

## 竞品分析与二次创作工作流

### ⚠️ 铁律：循环回避原则

**如果同一个方案失败 2 次，必须立刻完全切换方案，不得继续调整迭代。**

本项铁律源于腾哥明确批评：「方法不行就换个，不要一直循环」「咱别循环开了关关了开了」。

**禁止行为（已踩过的坑）：**
- ❌ 反复调整 Chrome 启动参数尝试 CDP 连 Windows 浏览器（无法断定为啥端口不工作，5+ 次徒劳）
- ❌ 反复尝试不同的 headless 浏览器绕过抖音验证码（100% 被拦）
- ❌ 在同一个思路上修修补补超过 2 次
- ❌ 腾哥说「你如果搞不定，我给你找个帮手」后仍继续尝试 — 这是中断信号，必须立刻停止当前方案
- ❌ 腾哥问「总失败是什么原因」— 这是对重复失败的明确不满，必须立刻切方案

**正确做法：**
- ✅ 第 1 次尝试 → 失败
- ✅ 第 2 次微调再试 → 仍失败
- ✅ **立即切换到完全不同的方案**，不要修修补补

### 方案选择优先级

| 优先级 | 方案 | 适用场景 |

### 通用借鉴原则

| 学习的底层技法 | 应用到全屋定制B端内容 |
|----------|---------------------|
| 用叙事让知识被"顺便"理解，而非硬塞 | 把「3个封边细节」改成「你知道为什么你的封边永远做不过大厂？」 |

| 不炫技，只通情绪 | 用大白话讲专业，比用行业术语更有信任感 |

### B端概念解释类脚本的交付标准（重要！腾哥多次修正后的最终版）

> 当腾哥的要求是「用全屋定制的方式解释一个概念给B端老板听」时，严格遵守以下结构。

#### 结构（不要打比方开头）

```
① 不是什么（排比否定） → 「SOP不是什么高大上的东西，也不是...」
② 是什么（用销冠/导购的具体场景） → 「你店里那个销冠，客户说考虑...」
③ 用精准数字的生活类比（番茄炒蛋/煮饭等） → 「好比番茄炒鸡蛋，番茄切3厘米块，油温160℃...」
④ 一句话总结 → 「这就是SOP」
```

#### 铁律（踩过的坑）

| 规则 | 说明 | 反例（被退回过） |
|------|------|-----------------|
| 开头不要「打比方」 | 先否定错误认知，再说它是什么 | ❌ 「SOP就像一本说明书」 |
| 例子必须精确到数字 | 有具体克数、分钟数、温度、次数才有说服力 | ❌ 「先炒蛋再炒番茄」→ ✅「油温160℃，先炒蛋30秒盛出，番茄2分钟出汁，合翻5下」 |
| 类比用厨房场景（腾哥钦定） | 番茄炒鸡蛋最易懂，不要用定制行业内部例子做类比 | ❌ 「好比装柜子，先装顶还是先装底」 |
| 主角用「销冠」不用「老师傅」 | 门店老板关心销售，不关心安装 | ❌「你店里那个老师傅」→ ✅「你店里那个销冠」 |
| 每条控制在20-25秒 | 腾哥反馈"有点长"后缩短 | ❌ 超过30秒的脚本 |
| 分清概念再解释 | 确认自己理解正确再写，比如SOP≠标准（标准=目的地，SOP=导航路线） | ❌ 混用SOP和标准 |
| 直给，不要悬念铺垫 | B端老板没耐心听铺垫 | ❌「你有没有想过一个问题…」 |

### CDP 主动搜索实战流程（2026-05-09 验证可用）

**搜索入口：** 抖音 SPA 的搜索框无法通过 `browser_type` + Enter 触发全局搜索（仅搜个人主页作品），**必须用 `browser_navigate` 直接跳转 URL**。

**URL 模式：**
```
# 搜索视频（推荐：信息最丰富）
https://www.douyin.com/search/{URL编码关键词}?type=video

# 搜索用户
https://www.douyin.com/search/{URL编码关键词}?type=user
```

**视频数据采集步骤：**

1. **搜视频结果** — `browser_navigate` 到视频搜索 URL，`browser_scroll(down)` 加载列表
2. **提取视频链接** — `browser_console` 执行：
   ```js
   document.querySelectorAll('a[href*="/video/"]')
   ```
   得到该搜索页所有视频的完整 URL
3. **逐条扒视频详情** — `browser_navigate` 到每个视频 URL：
   - 搜索结果已提供：标题、时长、播放量、Hashtag、@发布账号
   - 视频详情页可提取 **「章节要点」**（口播视频的内容大纲，含完整脚本骨架）
   - 章节要点提取方法：`browser_snapshot` 全量后搜索「章节要点」文本，或在页面上滚动到评论区上方区域查看

**搜索结果的解析（从搜索结果页的 link 元素获取）：**
```
link "00:42 3 报价的艺术 #全屋定制 #销售技巧... @洛阳鑫德 1周前"
# 说明：00:42=时长, 3=播放量(播放量<1000显示具体数字,≥1000显示1.2万等)
# 标题+Hashtag+@账号+发布时间在一个link内可直接读取
```

**关键词搜索策略（三重覆盖）：**

```

维度二：获客类 → 全屋定制门店获客引流 / 全屋定制获客培训 / 全屋定制短视频运营
```

### CDP 连接故障处理：WebSocket URL 过期

**问题：** `browser_navigate` 和 `browser_cdp` 工具会缓存上一个会话的 WebSocket URL（含 devtools browser UUID）。每次 Chrome 重启（因会话超时或手动重启）后，UUID 变化，导致工具报 404 错误。

**典型报错：**
```
WebSocket error talking to CDP at ws://172.28.32.1:9222/devtools/browser/<OLD_UUID>:
server rejected WebSocket connection: HTTP 404
```

**解决步骤（完整重连流程）：**

```bash
# 1. 杀 Chrome
powershell.exe "Get-Process chrome -ErrorAction SilentlyContinue | Stop-Process -Force"

# 2. 删旧转发
powershell.exe "netsh interface portproxy delete v4tov4 listenport=9222 listenaddress=0.0.0.0"

# 3. 启 Chrome（必须 user-data-dir，否则 profile 锁导致 --remote-debugging-port 被忽略）
powershell.exe "Start-Process -FilePath 'C:\Program Files\Google\Chrome\Application\chrome.exe' -ArgumentList '--remote-debugging-port=9222', '--user-data-dir=C:\temp\chrome-debug'"
sleep 6   # 等 Chrome 完全启动

# 4. 加新转发
WSL_GW=$(ip route show default | awk '{print $3}')
powershell.exe "netsh interface portproxy add v4tov4 listenport=9222 listenaddress=0.0.0.0 connectport=9222 connectaddress=127.0.0.1"
sleep 2

# 5. 验证新 UUID
curl -s "http://${WSL_GW}:9222/json/version"
```

**备用方案（当 browser_navigate 仍不工作时）：**
使用 Python + websockets 直连 CDP，绕过工具层的 URL 缓存：

```python
import json, asyncio, websockets

WS_URL = "ws://172.28.32.1:9222/devtools/browser/<CURRENT_UUID>"

async def extract_douyin_video(url):
    async with websockets.connect(WS_URL, max_size=10*1024*1024) as ws:
        # 创建新 tab
        await ws.send(json.dumps({"id": 1, "method": "Target.createTarget", "params": {"url": url}}))
        resp = json.loads(await ws.recv())
        target_id = resp["result"]["targetId"]
        await asyncio.sleep(8)
        
        # Attach 到 target
        await ws.send(json.dumps({"id": 2, "method": "Target.attachToTarget", "params": {"targetId": target_id, "flatten": True}}))
        # 监听 attachedToTarget 事件获取 sessionId
        while True:
            msg = json.loads(await ws.recv())
            if msg.get("method") == "Target.attachedToTarget":
                session_id = msg["params"]["sessionId"]
                break
        
        # 提取页面文本
        await ws.send(json.dumps({"id": 3, "sessionId": session_id, "method": "Runtime.evaluate",
            "params": {"expression": "document.body.innerText.substring(0, 8000)", "returnByValue": True}}))
        while True:
            msg = json.loads(await ws.recv())
            if msg.get("id") == 3:
                return msg["result"]["result"]["value"]

asyncio.run(extract_douyin_video("https://v.douyin.com/XXXX/"))
```

**注意事项：**
- 从 json/version 获取最新 UUID，不要硬编码
- 等待页面加载至少 6-8 秒让抖音 SPA 渲染完成
- 提取内容在 `body.innerText` 中，包含：视频标题、AI 章节要点、话题标签、评论区、相关推荐
- 章节要点是口播视频的 AI 摘要，是分析视频结构的主要信息来源
